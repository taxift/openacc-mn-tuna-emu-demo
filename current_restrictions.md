# 現状の記述制約（2026/03/05現在）

## 使用できるディレクティブについて

### データ転送
- enter data
- exit data

のみ使えます。

なお、それぞれcopyin節, copyout節にしか対応していません。

### カーネル指定
- parallel loop

のみ使えます。
なお、

- present()節

は必須です。
また、ネストループでは

- collapse()節

を指定できます。
なお、collapse節以外のディレクティブでネストループの並列展開を指示することはできません。

その他、最内ループでのみ

- loop vector

が指定できます。
ただし、collapseの指定されたネストループに loop vectorを書くことはできません。

また、gang節とvector節の同時指定には未対応です。

## 次元分割ごとの変換ルールについて

1次元分割の場合
- 8192以下のサイズはすべて同じコードに変換される
- 8192より大きい場合、8192の倍数のサイズのみ変換可能（それ以外はエラー）

2次元分割の場合
- MN-Core無印の場合[128][64]、MN-Core2の場合[64][64]で分割される
- 各次元の値が上記以下である場合、上記の値を指定したときと同じコードに変換される
- 各次元の値が上記より大きい場合、上記の倍数でなければエラーになる

3次元分割の場合
- MN-Core無印の場合[16][16][32]、MN-Core2の場合[16][16][16]で分割される
- 各次元の値が上記以下である場合、上記の値を指定したときと同じコードに変換される
- 各次元の値が上記より大きい場合、上記の倍数でなければエラーになる

なお、いずれの場合も実際にPE上に載るサイズかどうかは検証しません。

### 次元とループ変数のネストの順番が一致しない場合
Fortranは未実装につき、C言語にのみ言及。

以下のコード例のように、1次元目（一番左）から順番にループ変数が最外から内側へ対応していくようなコードになっていない場合には未対応。  
```C
#pragma acc parallel loop collapse(3)
for(i=0;i<N;i++)
    for(j=0;j<N;j++)
        for(k=0;k<N;k++)
            a[k][j][i]++;
```
ループ変数が上記の順の場合、必ず参照コードは[i][j][k]の順になっていないといけない。

## サイズの指定
- 演算子が入るパターンは未対応なので、ちゃんと数値で指定しないといけない

## 分割コンパイル
未対応。
